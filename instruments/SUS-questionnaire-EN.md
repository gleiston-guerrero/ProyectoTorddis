# System Usability Scale (SUS) — English reference version

The System Usability Scale was originally developed by John Brooke (1996). It is administered as a 10-item Likert-scale questionnaire, with each item rated from **1 = Strongly disagree** to **5 = Strongly agree**.

Odd-numbered items are worded positively; even-numbered items are worded negatively. The SUS score is computed as follows:

- For every odd-numbered item, subtract 1 from the score.
- For every even-numbered item, subtract the score from 5.
- Sum the transformed scores (range 0–40) and multiply by 2.5 to obtain the SUS score on a 0–100 scale.

The qualitative interpretation of the score follows Bangor et al. (2008):

| Range | Adjective rating |
|-------|------------------|
| < 25  | Worst imaginable |
| 25–51 | Poor             |
| 52–72 | Acceptable       |
| 73–85 | Good             |
| 86–100 (< 100) | Excellent   |
| = 100 | Best imaginable  |

## Items

| # | Statement | Scale |
|---|-----------|-------|
| 1 | I think that I would like to use this system frequently. | 1–5 |
| 2 | I found the system unnecessarily complex. | 1–5 |
| 3 | I thought the system was easy to use. | 1–5 |
| 4 | I think that I would need the support of a technical person to be able to use this system. | 1–5 |
| 5 | I found the various functions in this system were well integrated. | 1–5 |
| 6 | I thought there was too much inconsistency in this system. | 1–5 |
| 7 | I would imagine that most people would learn to use this system very quickly. | 1–5 |
| 8 | I found the system very cumbersome to use. | 1–5 |
| 9 | I felt very confident using the system. | 1–5 |
| 10 | I needed to learn a lot of things before I could get going with this system. | 1–5 |
